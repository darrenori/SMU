# Privacy model

## Data flow

1. The user enters a task in the Chrome popup.
2. Text is normalised locally.
3. Transparent keyword scoring chooses the closest fixed workflow.
4. A fixed response is formatted in the popup.
5. History is stored in the current browser profile.

There is no network step.

## Stored locally

- matter name, reference, client label and working notes;
- conversation history;
- checklist completion; and
- theme preference.

## Not collected

- identity or account information;
- usage analytics;
- browsing history;
- page content;
- credentials; and
- device or advertising identifiers.

The extension requests no Chrome permissions and cannot read the active webpage.
